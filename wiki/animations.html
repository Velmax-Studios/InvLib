Fluid Animations

Add life and movement to your interfaces with zero performance cost.

How it Works

InvLib uses a global AnimationTicker that manages all AnimatedButtons. To stay lightweight, the ticker only processes menus that are actively being viewed by at least one player.

Functional Animation

Use a lambda to define how the item looks based on the current tick.

ItemAnimation cycle = tick -> {
    Material mat = (tick % 20 < 10) ? Material.GOLD_INGOT : Material.IRON_INGOT;
    return new ItemStack(mat);
};

AnimatedButton button = new AnimatedButton(cycle);
menu.setButton(13, button);

Frame-based Animation

If you have a fixed sequence of items, use the frames() helper.

List<ItemStack> frames = List.of(frame1, frame2, frame3);
ItemAnimation anim = ItemAnimation.frames(5, frames); // 5 tick delay

Performance Fact

"The AnimationTicker uses WeakReferences to track menus, ensuring that closed or discarded menus are never ticked and don't leak memory."
